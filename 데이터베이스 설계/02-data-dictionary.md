# 데이터 사전 — 도서관 대출 관리 시스템

> 설계 둘째 산출물. 요구사항에 나온 **용어를 표준화**하고, 저장할 **데이터 항목**을 또렷이 정의한다.
> "사람마다 다르게 부르는 말"을 하나로 맞추는 단계 — 이게 흔들리면 뒤(ERD·표)가 다 흔들린다.

## 1. 용어 정의 (같은 것을 같은 말로)

| 용어 | 정의 |
|------|------|
| 회원(Member) | 도서관에 가입해 책을 빌릴 수 있는 사람. |
| 도서(Book) | 출판된 *책 한 종류*(제목·저자·ISBN 단위). "어떤 책인가". |
| 사본(Copy) | 같은 도서의 *물리적 한 권*. 인기 도서는 사본이 여러 권일 수 있다. *실제로 빌려가는 것은 '사본'*. |
| 저자(Author) | 책을 쓴 사람. 한 책에 여러 명, 한 명이 여러 책. |
| 대출(Loan) | 한 회원이 한 사본을 빌린 *사건*. 대출일·반납예정일·반납일을 가진다. |
| 연체(Overdue) | 반납예정일이 지났는데 아직 반납(반납일 없음)되지 않은 대출 상태. |

> **★ "도서" vs "사본" 구분이 핵심.** "해리포터"는 *도서 하나*지만, 도서관에 *3권(사본 3개)* 있을 수 있다. 대출은 그 3권 중 *특정 한 권*에 일어난다. 이 구분을 지금 잡아두면 뒤 설계가 깔끔하다.

## 2. 데이터 항목 (무엇을 어떤 형식으로 저장)

### 회원(Member)
| 항목 | 설명 | 형식(개념) | 규칙 |
|------|------|-----------|------|
| 회원번호 | 회원 식별자 | 정수(자동) | 유일·필수 |
| 이름 | 회원 이름 | 문자열 | 필수 |
| 이메일 | 연락·식별 | 문자열 | **유일**·필수 (C-1) |
| 전화번호 | 연락처 | 문자열 | 선택 |
| 가입일 | 가입 날짜 | 날짜 | 필수 |

### 도서(Book)
| 항목 | 설명 | 형식(개념) | 규칙 |
|------|------|-----------|------|
| 도서번호 | 도서 식별자 | 정수(자동) | 유일·필수 |
| ISBN | 국제 표준 도서번호 | 문자열 | **유일**·필수 (C-6) |
| 제목 | 책 제목 | 문자열 | 필수 |
| 출판사 | 펴낸 곳 | 문자열 | 선택 |
| 출판일 | 펴낸 날짜 | 날짜 | 선택 |
| 카테고리 | 분류(소설·과학…) | 문자열/코드 | 선택 |

### 저자(Author)
| 항목 | 설명 | 형식(개념) | 규칙 |
|------|------|-----------|------|
| 저자번호 | 저자 식별자 | 정수(자동) | 유일·필수 |
| 이름 | 저자 이름 | 문자열 | 필수 |

### 사본(Copy)
| 항목 | 설명 | 형식(개념) | 규칙 |
|------|------|-----------|------|
| 사본번호 | 사본 식별자 | 정수(자동) | 유일·필수 |
| (소속 도서) | 어떤 도서의 사본인지 | 도서 참조 | 필수 |
| 보관 위치 | 서가 위치 | 문자열 | 선택 |

### 대출(Loan)
| 항목 | 설명 | 형식(개념) | 규칙 |
|------|------|-----------|------|
| 대출번호 | 대출 식별자 | 정수(자동) | 유일·필수 |
| (대출 회원) | 누가 빌렸는지 | 회원 참조 | 필수 |
| (대출 사본) | 어떤 사본을 빌렸는지 | 사본 참조 | 필수 |
| 대출일 | 빌린 날 | 날짜 | 필수 |
| 반납예정일 | 반납 기한 | 날짜 | 필수 (= 대출일+14, C-3) |
| 반납일 | 실제 반납한 날 | 날짜 | 선택(아직 안 냈으면 비어 있음=대출 중) |

## 3. 관계(미리 보기 — ERD에서 확정)

- 회원 1 — N 대출 (한 회원이 여러 번 대출)
- 사본 1 — N 대출 (한 사본이 시간차로 여러 번 대출됨)
- 도서 1 — N 사본 (한 도서에 여러 물리 사본)
- 도서 N — M 저자 (다대다 → 다음 단계에서 연결 테이블 등장)

> 이 데이터 사전을 바탕으로, 다음 차시에서 **엔티티-관계 다이어그램(ERD)** 을 그린다.
